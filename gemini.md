## 1. Data Schemas

### Core Interaction
- **Input**:
  - `user_requirement`: String (The raw text from the user describing the feature to test)
  - `system_prompt`: String (The fixed "proper template" stored in code)
- **Output**:
  - `test_cases`: String (Markdown formatted test cases generated by Llama 3.2)

### API Structure (Draft)
- `POST /api/generate`
  - Body: `{ prompt: string }`
  - Response: `{ result: string }`

## 2. Behavioral Rules
- Prioritize reliability over speed.
- Never guess business logic.
- Follow B.L.A.S.T. protocol.
